# data/

This directory holds all pipeline data files.

## Subdirectories

### `samples/`
Contains sample outputs from the crawling and NER pipeline, committed to the
repository for reference and quick testing:

| File | Description |
|------|-------------|
| `samples/crawler_output.jsonl` | Sample JSONL output from the Wikipedia crawler (10 seed articles) |
| `samples/extracted_knowledge.csv` | Sample NER output: entities with labels and context |
| `samples/candidate_triples.csv` | Sample relation extraction output: subject-relation-object triples |

## Files generated at runtime (not tracked in git)

The following files are generated when the pipeline runs and are excluded from
version control due to their size:

| File | Generated by | Description |
|------|-------------|-------------|
| `crawler_output.jsonl` | `python src/crawl/run_lab1.py` | Full crawler output (~80 pages) |
| `extracted_knowledge.csv` | `python src/crawl/run_lab1.py` | Full NER output |
| `candidate_triples.csv` | `python src/crawl/run_lab1.py` | Full relation extraction output |
| `kge/train.txt` | `python src/kge/prepare_data.py` | KGE training split (80%) |
| `kge/valid.txt` | `python src/kge/prepare_data.py` | KGE validation split (10%) |
| `kge/test.txt` | `python src/kge/prepare_data.py` | KGE test split (10%) |
| `kge/entities.txt` | `python src/kge/prepare_data.py` | Entity dictionary |
| `kge/relations.txt` | `python src/kge/prepare_data.py` | Relation dictionary |

## How to regenerate

**Step 1: Crawl and extract entities/relations**
```bash
python src/crawl/run_lab1.py
```
This produces `data/crawler_output.jsonl`, `data/extracted_knowledge.csv`, and
`data/candidate_triples.csv`.

**Step 2: Build and expand the Knowledge Base**
```bash
python src/kg/run_td4.py
```
This consumes the CSV files above and produces the KG artifacts in `kg_artifacts/`.

**Step 3: Prepare KGE splits**
```bash
python src/kge/prepare_data.py
```
This reads `kg_artifacts/medical_kb_expanded.nt` and writes the train/valid/test
splits to `data/kge/`.
